# AdminBroadcast 模块功能说明

## 功能概述
- 管理员系统广播（系统级通知）。
- 统一管理员权限校验与广播参数校验。
- 服务器配置与运行时间模板管理。
- 超级管理员测试工具（创建测试用户、给指定用户创建随机策略并切换 `testing`、批量创建测试策略并切 testing）。
- 管理员策略接口：`admin.strategy.running.list` 获取运行中策略分页列表；`admin.strategy.running.by-market` 按市场聚合运行策略（交易所/币种/周期、策略数；不再依赖 `strategy_task_trace_log` 大表）；`admin.strategy.running.list-by-market` 按市场筛选运行策略（支持分页、搜索）。
- 管理员任务追踪接口：`admin.strategy.task.trace.list` 获取指定服务器实盘任务链路追踪分页列表；`admin.strategy.task.trace.layout` 获取按市场聚合布局（已切换为 `strategy_engine_run_log` 主记录模式，不再扫描 `strategy_task_trace_log`）；`admin.strategy.task.trace.market-summary` 获取指定市场的任务执行报告（主记录模式，返回任务数、平均耗时、成功率、分段统计、最近订单样本、最近5次任务样本）；`admin.strategy.run.metrics.recent` 获取指定策略最近运行画像（默认 5 条）。
- 超级管理员回测压测工具（批量创建随机策略 + 并发提交回测任务，输出性能统计快照）。

## 主要入口
- 控制器：`Controllers/AdminBroadcastController.cs`、`Controllers/AdminServerController.cs`、`Controllers/AdminPositionRiskController.cs`
- 配置与模板：`Controllers/AdminServerConfigController.cs`、`Controllers/AdminRuntimeTemplateController.cs`
- 用户管理与超级测试台：`Controllers/AdminUserController.cs`
- 服务：`Application/AdminBroadcastService.cs`

## 联动关系
- `Accounts`：通过 `AccountRepository` 校验管理员角色。
- `Notifications`：使用 `INotificationPublisher` 发送系统通知。
- `StrategyManagement`：通过 `StrategyRepository` 创建策略并切换策略实例状态。

## 超级测试台
### 权限与开关
- 仅超级管理员可访问（角色值由 `BusinessRules.SuperAdminRole` 控制，默认 `255`）。
- 功能总开关：`BusinessRules.EnableSuperAdminTestTools`（默认 `false`）。
- 当前端调用能力接口时，返回是否启用，便于前端隐藏入口。

### 接口
1. `POST /api/admin/user/test/capability`
- type: `admin.user.test.capability`
- 作用：返回超级测试台开关状态。
- 返回：`enabled: boolean`

2. `POST /api/admin/user/test/create-user`
- type: `admin.user.test.create-user`
- 入参：`email`、`password(>=6)`、`nickname?`
- 作用：创建普通角色测试账号（`role=0`），密码使用 BCrypt 哈希。
- 返回：`uid/email/nickname/role/status`

3. `POST /api/admin/user/test/create-random-strategy`
- type: `admin.user.test.create-random-strategy`
- 入参：
  - `targetUid`
  - `autoSwitchToTesting?`（默认 `true`）
  - `conditionCount?`（开多/开空条件数量，默认 5，范围 1～20）
  - `preferredExchange?`（可选，留空随机；支持：`binance|okx|bitget`）
  - `preferredTimeframeSec?`（可选，留空随机；支持：`60|300|900|3600|14400`）
  - `preferredSymbol?`（可选，留空随机；支持：`BTC/USDT|ETH/USDT|SOL/USDT|XRP/USDT`）
- 作用：
  - 使用全量 talib 指标池随机生成策略配置并创建到目标用户。
  - 支持按需固定交易所/时间周期/交易对，避免策略样本过于分散。
  - 创建后尝试切换策略实例到 `testing`。
  - 若切换失败，不回滚已创建策略，返回失败详情用于继续调试。
- 返回：
  - `strategy`: `defId/usId/versionId/versionNo/name/state`
  - `random`: `entryMethod/exitMethod/leftIndicator/rightIndicator/timeframe/exchange/symbol`
  - `testing`: `requested/success/message/state`

4. `POST /api/admin/user/test/delete-test-strategies`
- type: `admin.user.test.delete-test-strategies`
- 入参：`targetUid`（必填）
- 作用：一键删除目标用户下所有超级测试台创建的测试策略（匹配 description 为「超级管理员测试台随机策略」「超级管理员批量创建测试策略」「超级管理员回测压测自动生成策略」）。
- 返回：`targetUid/deleted/failed/elapsedMs/failures`

5. `POST /api/admin/user/test/batch-create-testing-strategies`
- type: `admin.user.test.batch-create-testing-strategies`
- 入参：`targetUid`（必填）、`strategyCount`（默认 10，最大 1000）、`conditionCount?`（默认 5，范围 1～20）、`preferredExchange/preferredSymbol/preferredTimeframeSec?`
- 作用：批量创建随机策略并切换 testing。
- 返回：`targetUid/requested/created/failed/switchSuccess/switchFailed/elapsedMs/failures`

6. `POST /api/admin/user/test/backtest-stress`
- type: `admin.user.test.backtest-stress`
- 入参：
  - `targetUid`（必填，目标测试用户 UID）
  - `strategyCount`（默认 20，最大 500）
  - `conditionCount?`（开多/开空条件数量，默认 5，范围 1～20）
  - `tasksPerStrategy`（默认 1，最大 20）
  - `submitParallelism`（默认 4，最大 64）
  - `barCount`（默认 1500，范围 100~500000）
  - `initialCapital`（默认 10000）
  - `executionMode`（`batch_open_close|timeline`）
  - `includeDetailedOutput`（是否输出交易/曲线/事件明细）
  - `pollAfterSubmitSeconds`（提交后等待秒数，默认 5，最大 120）
  - `preferredExchange/preferredSymbol/preferredTimeframeSec`（可选，复用随机策略参数）
- 作用：
  - 批量创建随机策略（落到目标用户）。
  - 并发调用回测任务提交链路，执行回测提交压测。
  - 返回创建耗时、提交延迟分位（P50/P95/P99）、提交成功率、任务状态分布、线程池与进程线程快照。
- 返回：
  - `requested`：本次压测请求参数与总任务数。
  - `strategies`：策略创建成功/失败统计、创建耗时与样本。
  - `submissions`：任务提交成功/失败、提交延迟统计、参与线程信息、失败样本。
  - `taskStatus`：提交后即时状态分布与等待后状态分布。
  - `runtime`：压测前/提交后/等待后的运行时快照（线程池、内存、线程数）。

### 随机策略说明
- 指标来源：`Config/talib_indicators_config.json` + `talib_web_api_meta.json`（优先 `Config/`，回退 `Client/public/`）。
- 指标抽样范围：默认限制为常用指标池（如 `MACD/RSI/MA/EMA/SMA/BBANDS/ATR/ADX/CCI/STOCH/MFI/OBV/WILLR/ROC/MOM`），不再从全量 160 指标中无约束抽样。
- 参数来源：优先 meta 默认值，缺失时回退 common 默认值，`Integer/MAType` 按整数处理。
- 条件方法池：`GreaterThan/LessThan/CrossUp/CrossDown/CrossAny`，退出条件使用反向方法。
- 策略结构：`entry.long` 与 `entry.short` 各 N 个条件，`exit.long` 与 `exit.short` 各 N 个条件，需全部满足才触发，N 由 `conditionCount` 控制（默认 5，范围 1～20），`runtime.scheduleType=Always`。
