# Monitoring 模块功能说明

## 功能概述
- 提供系统启动监控宿主抽象与运行状态日志采集能力。
- 主服务默认使用无 UI 空实现，避免对 Windows/WinForms 产生硬依赖。
- 汇总关键模块的启动与运行状态，便于运维与排障。
- 提供协议性能监控能力，统一记录 HTTP / WebSocket 协议交互的前端总耗时、后端处理耗时与异常分布，用于性能分析与数据优化。

## 配置选项
- `MonitoringOptions.EnableDesktopHost`：是否尝试启用进程内桌面监控（仅 Windows 有效，默认 `false`）。
- `MonitoringOptions.EnableProtocolPerformanceStorage`：是否启用协议性能数据采集与 MySQL 入库，默认 `false`；关闭时不记录 HTTP / WebSocket 协议耗时，也不创建/写入协议性能监控表。
- `MonitoringOptions`：监控窗口日志与交易日志的缓存上限。

## 主要入口
- 监控宿主接口与实现：`Application/IStartupMonitorHost.cs`、`Application/NoopStartupMonitorHost.cs`、`Application/WinFormsStartupMonitorHost.cs`
- 监控窗体相关代码（保留，主服务默认不编译）：`Application/StartupMonitorHost.cs`、`Application/StartupMonitorForm.cs`
- 日志提供器：`Infrastructure/StartupMonitorLoggerProvider.cs`
- 领域模型：`Domain/StartupMonitorLogEntry.cs`
- 协议性能采集：`Application/ProtocolPerformance*`、`Infrastructure/ProtocolPerformance*`、`Domain/ProtocolPerformance*`

## 协议性能监控说明
- 采集范围：HTTP 协议请求、WebSocket 请求应答（心跳 `ping/pong` 不纳入统计）。
- 关联键：统一使用 `reqId + transport` 作为前后端数据合并键。
- 服务端口径：
  - HTTP：记录请求进入服务到响应完成的服务端耗时。
  - WebSocket：记录消息解析完成后到应答写回完成的服务端耗时。
- 前端口径：
  - HTTP：记录从发起请求到收到响应的总往返耗时。
  - WebSocket：记录从发送请求消息到收到对应 `reqId` 应答的总往返耗时。
- 入库方式：
  - 服务端在协议链路完成后异步写入服务端耗时。
  - 前端将耗时批量上报到后端，由后端按相同 `reqId` 合并更新。
  - 仅当 `Monitoring.EnableProtocolPerformanceStorage=true` 时才会启用采集与入库。
- 输出能力：
  - 提供协议汇总统计接口，按协议类型输出次数、均值、峰值、慢请求占比与错误率。
  - 提供异常分析接口，按固定阈值和相对偏离度识别耗时过长协议。

## 联动关系
- `MarketData`：展示历史行情缓存与下载状态。
- `MarketStreaming`：展示行情流与连接状态（通过 `IConnectionManager`）。
- `StrategyEngine`：展示策略引擎相关状态（由启动管理器收集）。
- `Shared`：复用统一协议上下文、数据库访问与异常处理中间件。
- `WebSockets`：复用统一消息分发入口，集中记录单条 WS 协议耗时。
- `Client/src/network`：统一采集 HTTP / WS 前端总耗时并批量上报。
