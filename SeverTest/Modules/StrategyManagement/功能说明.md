# StrategyManagement 模块功能说明

## 功能概述
- 策略定义、版本与配置管理（增删改查）。
- 面向用户的策略管理入口。
- 运行时间配置校验（自定义按小时精度，模板可分钟；模板仅保存模板 ID）。
- 运行时间模板列表查询（数据库模板库 + 缓存）。
- 策略创建工作台升级为全屏布局：
  - 自定义创建前增加确认弹窗，确认后跳转全屏工作台。
  - 工作台使用 Portal 独立挂载（脱离策略主容器），覆盖主界面且层级低于公共 Dialog。
  - 顶部返回/设置统一入口。
  - 进入后先在居中面板选择交易所/币种/周期（默认币安 + 5 分钟），再展开工作区。
  - 当用户“新建策略”且没有 `initialConfig` 时，工作台会自动注入默认示例策略：`EMA(15)` 上穿 `SMA(50)` 触发开多；开空/平多/平空容器保持启用但为空，避免首屏空白。
  - 左侧 K 线（后端历史K线接口）+ 本地回测仪表盘，右侧指标/操作符/条件容器分区编辑。
  - 图表与已选指标联动：新增/修改/移除指标时，工作台 K 线会自动同步对应 TA 指标及参数显示。
  - 工作台“仓位列表”点击联动：点击任一仓位后，K 线图会聚焦该仓位时间区间，并在图上绘制止盈止损风险收益区间（`riskRewardLong/riskRewardShort`）。
  - 工作台“仓位列表”改为全量展示，不再仅显示前 6 条；列表项启用浏览器可见性优化，降低长列表渲染压力。
  - 工作台“仓位列表”新增“全量预览”开关：
    - 普通预览：保持“点击某条仓位 -> 图表聚焦单笔仓位并绘制单笔风险收益区间”。
    - 全量预览：列表滚动与 K 线可视区双向联动（滚动任一侧，另一侧同步定位）；K 线按当前可视范围批量绘制仓位风险收益区间（仅绘制可见窗口内仓位，并限制数量上限以保证流畅性）。
  - 工作台“仓位列表”新增“联动开关（图标+Toggle）”：用于控制“滚动仓位列表 -> K线跟随”与“拖动K线 -> 列表跟随”两条滚动联动链路，用户可按需启停联动。
  - 工作台“联动开关”默认关闭，用户可按需主动开启联动。
  - 工作台图表中的仓位风险收益区（`riskRewardLong/riskRewardShort`）强制锁定：不允许拖动，右键不允许删除，避免误操作破坏回测预览。
  - 本地实时检测前置门槛：止盈、止损、杠杆、单次开仓量未完成时不启动检测。
- 条件合法性双端校验（前端实时 + 后端强校验）：
  - 指标周期必须与策略交易周期一致（单策略单周期）。
  - 禁止主图/副图指标直接比较。
  - 禁止跨周期指标直接比较（例如 1m 与 5m 的 SMA 配对判断）。
  - 禁止数字与数字直接比较。
  - 禁止同一指标（含同参数同输出）或同一字段与自身比较。
  - 允许 K 线字段互相比较（如 Open vs Close），但仍禁止同一字段与自身比较。
  - 限制条件规模：每组最多 6 条、每集合最多 3 组、总条件最多 108 条，防止垃圾配置与条件风暴。
- 策略状态支持 `testing`：切换到测试状态时不再强制绑定 API Key，并清空实例绑定 Key，转入本地模拟执行路径。
- `strategy.delete` 成功后立即执行运行时卸载与租约释放，避免策略在删除后继续执行。
- `strategy.delete` 增加“动作封禁→停机卸载→释放租约→删除记录”顺序，任一步骤异常将返回失败，不再吞异常后仍返回成功。
- `strategy.delete` 增加兜底异常回滚封禁：删除流程在外层异常分支也会自动解除动作封禁，避免实例残留但被永久封禁。
- 策略绩效缓存服务（`StrategyPerformanceCacheService`）：
  - 统一输出 30 日资金曲线（优先实盘）。
  - 当 30 日无实盘交易时自动触发回测补齐。
  - 对公开帖子/分享码/公开市场/官方策略复用缓存，避免重复回测占用 CPU。
  - 实盘指纹未变化时直接续期缓存，并复用历史曲线与日志快照，减少重复聚合。
  - 无实盘时按“策略内容哈希 + 窗口天数 + UTC日期”复用回测缓存，避免同日重复回测。
  - 缓存中落地交易日志、仓位日志、开平事件日志（`trade_log_json/position_log_json/open_close_log_json`）。
  - `strategy.official.list`、`strategy.market.list`、`strategy.share.create` 统一返回绩效字段：
    - `pnlSeries30d`、`curveSource`、`isBacktestCurve`。
  - 公开动作（官方发布/同步、市场发布/同步、创建分享码）后自动预热缓存，降低首访延迟。

## 主要入口
- 控制器：`Controllers/StrategyController.cs`
- 服务/仓储：`Application/StrategyService.cs`、`Application/StrategyPerformanceCacheService.cs`、`Infrastructure/StrategyRepository.cs`

## 联动关系
- `Accounts`：策略归属与权限校验。
- `ExchangeApiKeys`：策略与交易所 API Key 绑定信息。
- `StrategyEngine`：策略内容与校验逻辑。
- `StrategyRuntime`：运行时加载与状态管理。
- `Backtest`：无实盘窗口时用于资金曲线回测补齐。

## 前端联动（StrategyDetailDialog）
- “策略状态”区域新增“测试中”按钮，可直接将实例切换到 `testing`。
- 回测页签的“回测结果”新增全屏专用工作台入口（按钮：`backtest-fullscreen-btn`）。
- 全屏工作台布局：
  - 左侧：`MarketChart` 行情图表组件（根据回测参数/结果映射 `exchange:symbol` 与 `interval`）。
  - 右侧：回测结果明细（汇总统计、按标的交易明细、资金曲线、事件日志）。
- 全屏工作台为弹层模式，支持遮罩点击和 `Esc` 退出，不再使用“结果卡片直接固定定位撑满屏幕”的方式。
- 交易明细联动图表：
  - 在回测结果的“交易明细”表格中点击某条仓位，会触发图表焦点联动。
  - 打开“量化回测专用界面”时，若当前未选中仓位，默认自动聚焦“最新开仓”的一笔仓位（按 `entryTime` 最近优先），减少首次进入后的手动定位操作。
  - 联动流程为：前端根据仓位开平仓时间发起 `marketdata.kline.history` 请求（附带前后缓冲区间）→ `MarketChart` 切换并渲染该区间 K 线。
  - 图中会标记开仓/平仓时点与仓位区间，帮助用户快速定位该仓位的触发与演变过程。
  - 图表指标支持按策略配置自动预设：从策略逻辑中的指标引用自动提取指标代码（如 `EMA`、`MA`、`MACD`），进入回测专用界面后仅加载这些策略相关指标，避免无关默认指标干扰。
  - 指标预设会同步应用策略内配置的可控参数：`params`（指标参数）与 `input`（输入源），进入回测专用界面后不再回退到指标默认参数。
  - 未平仓仓位可视化增强：当仓位无平仓时间时，图表联动会以“最新数据时间 + 10 根K线”作为可视化终点，确保仍可绘制并明确展示“未平仓”状态。
  - 回测后端已配合调整：样本结束不再强制平仓，未完成仓位会以 `isOpen=true` 快照返回；前端按该标记展示“未平仓”并沿用未平仓联动绘制规则。
  - `MarketChart` 采用“数据版本号”保护历史加载与焦点加载，避免并发请求回写导致区间错位或普通历史数据覆盖焦点区间。
  - 焦点模式新增统一退出恢复：当 `focusRange` 被清空、symbol/interval 不匹配或焦点加载失败时，会清理焦点覆盖层并回退到常规历史窗口，恢复实时更新。
  - 焦点模式下关闭实时 tick 直推，避免在浏览历史仓位时被“当前最新 K 线”插入导致视图跳动。
  - 仓位焦点周期改为“首次对齐 + 后续放开”：
    - 点击新仓位时，会按仓位建议周期/可视上限自动选择一次初始周期。
    - 初始定位完成后，允许用户手动切换到 1m/5m/15m 等任意周期继续观察，同一仓位焦点与绘图区间不会被强制拉回原周期。
    - 焦点模式下忽略外部 `interval` 持续同步（仅首帧初始化生效），修复“手动切到 5m 后被立即拉回 1m”的问题。
  - 左右滑动分页改为“按已加载时间边界推进”：左滑使用当前最小时间戳向前补，右滑使用当前最大时间戳向后补，避免可视锚点漂移导致重复页与抖动。
  - 首次点击仓位时会额外预取更多右侧 K 线，降低“右侧空白立刻触发 Backward 补拉”的概率，减少首次进入焦点区间的连跳。
  - 焦点首屏增加短暂的 `Backward` 自动触发抑制窗口，仅用于过滤初始化阶段的边界补拉，避免首帧回跳；窗口结束后右滑补拉恢复正常。
  - 本地 `klinecharts` 已修正 `Backward` 追加行为：右侧增量数据追加后会保持当前可视锚点不变，避免每次补拉都把视图推向最新位置。
  - 分页请求失败时保持 `more=true`，允许用户继续右滑/左滑重试，避免一次网络抖动后分页能力被永久关闭。
  - 绘图工具图标渲染增加容错：当 SVG 资源未被 `svgr` 转换为 React 组件、仅返回 URL 字符串时，前端自动回退为 `<img>` 渲染，避免 `InvalidCharacterError` 导致页面崩溃。
