# AiAssistant 模块功能说明

## 模块职责
- 为前端聊天页提供 AI 对话接口。
- 将用户自然语言诉求转换为平台可用策略 JSON（`strategyConfig`）。
- 在模型输出后做服务端结构归一化，降低非法 JSON 进入业务链路的风险。
- 持久化保存用户会话与消息，支持多会话切换、跨设备续聊。

## 目录结构
- `Controllers/AiAssistantController.cs`：HTTP 入口，协议类型校验、用户鉴权与调用编排。
- `Application/AiAssistantConversationService.cs`：会话编排（建会话、拉历史、写消息、调用模型）。
- `Application/AiAssistantService.cs`：提示词拼装、知识库注入、模型结果解析与策略配置归一化。
- `Infrastructure/AiAssistantChatRepository.cs`：会话/消息持久化与建表。
- `Infrastructure/DeepSeekChatClient.cs`：DeepSeek(OpenAI 兼容) HTTP 调用封装。
- `Domain/AiAssistantContracts.cs`：请求/响应与历史消息模型。
- `Sql/`：AI 会话持久化 SQL 说明。

## 对外接口
- `POST /api/ai-assistant/conversations/list`
  - 协议类型：`ai.assistant.conversation.list`
  - 请求：`limit`（可选）
  - 返回：`items[]`（会话列表）
- `POST /api/ai-assistant/conversations/create`
  - 协议类型：`ai.assistant.conversation.create`
  - 请求：`title`（可选）
  - 返回：会话摘要（`conversationId/title/lastMessageAt` 等）
- `POST /api/ai-assistant/conversations/messages`
  - 协议类型：`ai.assistant.conversation.messages`
  - 请求：`conversationId`、`limit`（可选）
  - 返回：`conversation + messages[]`
- `POST /api/ai-assistant/chat`
  - 协议类型：`ai.assistant.chat`
  - 请求：`conversationId`、`message`
  - 返回：`conversationId`、`conversationTitle`、`reply`、`strategyConfig`

## 知识库内容（AI助手知识库.md）
- 策略 JSON 的 `trade` / `logic` / `runtime` 结构说明。
- `logic` 完整层级：containers → checks → groups → conditions，其中 `entry.long/entry.short` 的 groups 与 conditions 必须可执行。
- 条件 method 白名单与参数来源规则（args/param 的职责与兼容关系）。
- 条件 method 已补充 28 个检测方法的“代码级语义说明”（含 args/param 索引、周期/阈值回退逻辑、可复制示例）。
- 值引用（`StrategyValueRef`）字段规范：`refType/indicator/timeframe/input/params/output/offsetRange/calcMode`。
- 指标白名单（全量）：来自 `Config/talib_indicators_config.json` 的 160 个指标，逐项列出 inputs/options/outputs。
- 示例策略提供完整嵌套结构（含 entry/exit 四分支），避免仅写口语化条件导致不可落地。

## 关键流程
1. 控制器读取并校验请求、解析当前用户 `uid`。
2. 会话服务按需建表（`AiAssistant.AutoCreateSchema=true` 时自动执行）。
3. 根据 `conversationId` 读取历史消息；若为空则创建新会话。
4. 应用服务加载知识库文档（默认：`Document/AI助手知识库.md`）。
5. 组装 system prompt + 历史消息 + 当前消息，请求 DeepSeek。
6. 解析模型返回 JSON，提取 `assistantReply` 与 `strategyConfig`。
7. 若模型 HTTP 非 2xx，按状态码映射可读错误（如 402 映射为余额/额度问题）并返回前端。
8. 对 `strategyConfig` 执行默认值回填与结构修正，保证字段完整性。
9. 若模型返回了空条件（如 `groups=[]`），服务端会执行可执行兜底：
   - 能从用户语义提取 `EMAx` 与 `MAy/SMAy` 时，自动补 `CrossUp/CrossDown` 条件（EMA 对 SMA）；
   - 否则退化为 `Close` 与 `SMA20` 的交叉条件，确保策略可触发。
10. 持久化写入用户消息与助手消息，并更新会话摘要。
11. 通过统一协议包装返回前端。

## 配置项（appsettings.json）
- `AiAssistant.Enabled`：是否启用。
- `AiAssistant.AutoCreateSchema`：是否自动建表。
- `AiAssistant.BaseUrl`：模型服务地址。
- `AiAssistant.ApiKey`：模型 API 密钥。
- `AiAssistant.Model`：模型名称（如 `deepseek-chat`）。
- `AiAssistant.TimeoutSeconds`：请求超时秒数。
- `AiAssistant.Temperature`：采样温度。
- `AiAssistant.KnowledgeFilePath`：知识库文档路径。
- `AiAssistant.MaxHistoryMessages`：透传模型的最大历史消息条数。

## 联动模块
- `Accounts`：通过 `AuthTokenService` 做用户鉴权。
- `Shared/Infrastructure/Db`：通过 `IDbManager` 执行会话与消息持久化。
- `Shared/Protocol`：沿用统一协议请求与响应封装。
- `StrategyManagement / StrategyEngine`：复用策略 JSON 结构定义，保持可用性一致。
