# Indicators 模块功能说明

## 功能概述
- 对接 CoinGlass 等第三方指标源，统一采集、缓存、持久化与对外协议输出。
- 当前支持“HTTP 拉取 + WS 拉流缓存 + HTTP 对外查询”的统一底座。
- 开发阶段允许临时接入第三方非官方聚合商（盗版源）联调，正式上线前必须切换官方源。
- 提供可扩展的指标定义体系：新增指标优先通过“配置 + 采集器”方式扩展，避免重复造接口。
- 支持三层缓存链路：内存缓存（L1）+ Redis（L2）+ MySQL 快照（L3）。
- 支持历史点位存储与查询，满足前端图表展示、回放和横向对比。
- CoinGlass 实时缓存与 WS 桥接内部曾提供高频 Debug 级诊断日志（缓存命中、每条频道消息等），当前已在代码中统一注释，默认不输出，仅在联调 / 排查实时链路问题时可临时打开对应 LogDebug 调用。

## 当前内置指标
- `coinglass.fear_greed`：CoinGlass 贪婪恐慌指数（`shape=gauge`）。
- `coinglass.etf_flow`：CoinGlass 现货 ETF 净流入（`shape=timeseries`，支持 `BTC / ETH / SOL / XRP` 多资产 scope）。
- `coinglass.futures_footprint`：CoinGlass 合约足迹图（`shape=heatmap`，当前支持 `BTC / ETH`，默认 `Binance + 15m`）。
- `coinglass.top_long_short_account_ratio`：CoinGlass 大户账户数多空比（`shape=timeseries`，当前支持 `BTC / ETH`，默认 `Binance + 15m`）。
- `coinglass.grayscale_holdings`：CoinGlass 灰度持仓（`shape=table`）。
- `coinglass.coin_unlock_list`：CoinGlass 代币解锁列表（`shape=table`）。
- `coinglass.coin_vesting`：CoinGlass 代币解锁详情（`shape=table`，按 `symbol` scope 切换）。
- `coinglass.liquidation_heatmap_model1`：CoinGlass 交易对爆仓热力图（模型1，`shape=heatmap`）。
- `coinglass.exchange_assets`：CoinGlass 交易所资产明细（`shape=bar`，默认 `exchangeName=Binance`）。
- `coinglass.exchange_balance_list`：CoinGlass 交易所余额排行（`shape=bar`，默认 `symbol=BTC`）。
- `coinglass.exchange_balance_chart`：CoinGlass 交易所余额趋势（`shape=timeseries`，默认 `symbol=BTC`）。
- `coinglass.hyperliquid_whale_alert`：CoinGlass Hyperliquid 鲸鱼提醒（`shape=bar`）。
- `coinglass.hyperliquid_whale_position`：CoinGlass Hyperliquid 鲸鱼持仓（`shape=bar`）。
- `coinglass.hyperliquid_position`：CoinGlass Hyperliquid 持仓排行（`shape=bar`，按 `symbol` scope 切换）。
- `coinglass.hyperliquid_user_position`：CoinGlass Hyperliquid 用户持仓（`shape=table`，按 `userAddress` scope 切换）。
- `coinglass.hyperliquid_wallet_position_distribution`：CoinGlass Hyperliquid 钱包持仓分布（`shape=bar`）。
- `coinglass.hyperliquid_wallet_pnl_distribution`：CoinGlass Hyperliquid 钱包盈亏分布（`shape=bar`）。
- 贪婪恐慌采集器兼容两类上游结构：对象数组点位（如 `[{ts,value}]`）与列式数组（如 `data.values + data.dates`）。
- ETF 净流入采集器兼容 `data` 包裹与数组直出结构，统一抽取 `timestamp + flow_usd` 作为点位，并按 `scope.asset` 路由到 BTC / ETH / SOL / XRP 对应接口。
- 合约足迹图采集器兼容 `data[]` 二维数组结构，统一抽取 `timestamp + bins[]`，并聚合输出净主动买卖额、成交笔数与最新价格桶分布。
- 大户账户数多空比采集器兼容 `data[]` 结构，统一抽取 `time + top_account_long_percent + top_account_short_percent + top_account_long_short_ratio`，并按 `scope.asset` 路由到 BTC / ETH。
- 灰度持仓采集器兼容 `data[]` 列表结构，统一输出 `items[] + totalHoldingsUsd + premiumPositiveCount/premiumNegativeCount`。
- 代币解锁列表采集器兼容 `data/list/items/coins/rows` 等数组结构，统一输出 `items[] + totalCoinCount + totalMarketCap + totalNextUnlockValue`，用于数据库快照缓存与前端列表展示。
- 代币解锁详情采集器兼容 `data/result/item` 等对象结构，并宽松解析 `allocationItems[] + scheduleItems[] + supply/nextUnlock` 字段，按 `scope.symbol` 写入历史表，降低重复请求外部接口的频率。
- 爆仓热力图采集器兼容 `data` 包裹结构，统一输出 `yAxis`、`liquidationLeverageData` 与 `priceCandlesticks` 三段结构。
- 交易所资产明细采集器兼容官方 `data[]` 列表，统一输出 `items[] + totalBalanceUsd`。
- 交易所余额排行采集器兼容官方 `data[]` 列表，统一输出 `items[] + totalBalance`。
- 交易所余额趋势采集器兼容 `timeList + dataMap + priceList` 结构，统一输出时间轴、价格线与交易所余额序列。
- Hyperliquid 鲸鱼提醒采集器兼容官方 `data[]` 列表，统一输出 `items[] + totalPositionValueUsd + longAlertCount + shortAlertCount`。
- Hyperliquid 鲸鱼持仓采集器兼容官方 `data[]` 列表，统一输出 `items[] + totalPositionValueUsd + totalMarginBalance`。
- Hyperliquid 持仓排行采集器兼容 `data.total_pages/current_page/list[]` 结构，按 `scope.symbol` 路由并统一输出 `items[] + totalPositionValueUsd + totalPages + currentPage`。
- Hyperliquid 用户持仓采集器兼容 `data.margin_summary/cross_margin_summary/asset_positions[]` 结构，按 `scope.userAddress` 或 `scope.user_address` 路由并统一输出 `marginSummary + crossMarginSummary + assetPositions[]`。
- Hyperliquid 钱包持仓分布采集器兼容官方 `data[]` 列表，统一输出 `items[] + totalPositionUsd + totalPositionAddressCount + maxBiasScore`。
- Hyperliquid 钱包盈亏分布采集器兼容官方 `data[]` 列表，统一输出 `items[] + totalPositionUsd + totalPositionAddressCount + maxBiasScore`。

## 公共策略信号（策略可直接引用）
- 公共指标已支持派生信号输出，作为 `payload.signals.*` 字段长期稳定对外。
- 贪婪恐慌（`coinglass.fear_greed`）已内置：
  - `signals.below9`、`signals.below10`
  - `signals.below9StreakDays`、`signals.below10StreakDays`
  - `signals.below9Consecutive3d`、`signals.below10Consecutive3d`
- ETF 净流入（`coinglass.etf_flow`）已内置：
  - `asset`
  - `signals.isNetInflow`、`signals.isNetOutflow`
  - `signals.netInflowStreakDays`、`signals.netOutflowStreakDays`
  - `signals.netInflow3dAllPositive`
  - `signals.netFlow7dSumUsd`、`signals.netFlow7dSumPositive`
  - `signals.inflow7dUsd`、`signals.outflow7dAbsUsd`、`signals.inflow7dRatio`
- 合约足迹图（`coinglass.futures_footprint`）当前稳定输出：
  - `asset`、`exchange`、`symbol`、`interval`
  - `latestNetDeltaUsd`
  - `latestBuyUsd`
  - `latestSellUsd`
  - `latestTotalTradeCount`
  - `latestBins[]`
- 大户账户数多空比（`coinglass.top_long_short_account_ratio`）当前稳定输出：
  - `asset`、`exchange`、`symbol`、`interval`
  - `latestRatio`
  - `topAccountLongPercent`
  - `topAccountShortPercent`
- 以上字段会同时写入 latest 与 history，便于实时策略与回测统一复用。
- ETF 历史与快照继续复用同一指标编码 `coinglass.etf_flow`，通过 `scope_key=asset=BTC|ETH|SOL|XRP` 区分不同资产数据。
- 合约足迹图历史与快照复用同一指标编码 `coinglass.futures_footprint`，通过 `scope_key=asset=BTC|ETH` 区分不同资产数据。
- 大户账户数多空比历史与快照复用同一指标编码 `coinglass.top_long_short_account_ratio`，通过 `scope_key=asset=BTC|ETH` 区分不同资产数据。

## 可扩展字段目录（系统化输出）
- 指标定义表 `indicator_definitions.config_json` 作为“来源 + 字段目录”统一配置中心：
  - `source.type`：来源类型（示例：`http_pull`、`custom_compute`）。
  - `fields[]`：字段清单（`path/dataType/unit/conditionSupported/description`）。
- `indicator.meta.list` 会返回字段目录，客户端/策略构建器无需硬编码字段名。
- 默认 seed 采用“空配置自动补齐”策略：仅当已有 `config_json` 为空或 `{}` 时才回填，避免覆盖自定义配置。

## 缓存与刷新策略
- 查询顺序：内存缓存 -> Redis -> MySQL 快照 -> 第三方拉取。
- 数据过期后可返回 `stale=true` 的旧快照（受 `Indicators.StaleToleranceSeconds` 控制），并在后台异步刷新。
- 同一指标+scope 采用单飞锁（`SemaphoreSlim`），防止并发击穿第三方接口。
- 刷新成功后同时写入：
  - 最新快照表 `indicator_snapshots`
  - 历史分表 `coinglass_{指标代码}_history`（例如 `coinglass_fear_greed_history`、`coinglass_etf_flow_history`、`coinglass_futures_footprint_history`、`coinglass_top_long_short_account_ratio_history`、`coinglass_grayscale_holdings_history`、`coinglass_coin_unlock_list_history`、`coinglass_coin_vesting_history`、`coinglass_liquidation_heatmap_model1_history`、`coinglass_exchange_assets_history`、`coinglass_exchange_balance_list_history`、`coinglass_exchange_balance_chart_history`、`coinglass_hyperliquid_whale_alert_history`、`coinglass_hyperliquid_whale_position_history`、`coinglass_hyperliquid_position_history`、`coinglass_hyperliquid_user_position_history`、`coinglass_hyperliquid_wallet_position_distribution_history`、`coinglass_hyperliquid_wallet_pnl_distribution_history`）
  - 刷新日志表 `indicator_refresh_logs`

## 当前项目配置（appsettings）
- 贪婪恐慌：`FearGreedEnabled=false`；默认定义为禁用，刷新周期与 TTL 均为 `1200` 秒（20 分钟）。

## 配置项
- `Indicators`：
  - `Enabled`：是否启用指标框架
  - `AutoCreateSchema`：是否自动建表
  - `AutoSeedDefinitions`：是否自动写入默认指标定义
  - `RefreshScanIntervalSeconds`：后台刷新扫描周期
  - `DefinitionReloadSeconds`：定义缓存刷新周期
  - `MaxHistoryQueryPoints`：历史查询最大点位
  - `RedisCacheSeconds`：Redis 最短缓存时长
  - `StaleToleranceSeconds`：过期容忍时间
  - `HistoryCleanupIntervalMinutes`：历史清理扫描周期
- `CoinGlass`：
  - `SourceMode`：`official` / `pirated_proxy`（开发阶段临时使用）
  - `Enabled`：是否启用 CoinGlass
  - `BaseUrl`：API 基础地址
  - `ApiKey`：API Key
  - `ApiKeyHeaderName`：鉴权头名（官方 `CG-API-KEY` / 聚合商 `X-Api-Key`）
  - `RoutePrefix`：可选路径前缀（聚合商 V4 通常为 `v4`）
  - `FearGreedPath`：贪婪恐慌接口路径
  - `FearGreedSeriesLimit`：拉取点位上限
  - `EtfFlowPathTemplate`：ETF 净流入接口模板路径（推荐，如 `/api/etf/{asset}/flow-history`）
  - `EtfFlowPath`：ETF 净流入接口基础路径（未配置模板时作为 BTC 基线路径）
  - `EtfFlowSeriesLimit`：ETF 净流入保留点位上限
  - `FuturesFootprintPath`：合约足迹图接口路径
  - `FuturesFootprintDefaultExchange`：合约足迹图默认交易所（如 `Binance`）
  - `FuturesFootprintDefaultInterval`：合约足迹图默认粒度（当前固定 `15m`）
  - `FuturesFootprintSeriesLimit`：合约足迹图默认保留 K 线数量
  - `TopLongShortAccountRatioPath`：大户账户数多空比接口路径
  - `TopLongShortAccountRatioDefaultExchange`：大户账户数多空比默认交易所（如 `Binance`）
  - `TopLongShortAccountRatioDefaultInterval`：大户账户数多空比默认粒度（当前固定 `15m`）
  - `TopLongShortAccountRatioSeriesLimit`：大户账户数多空比保留点位上限
  - `GrayscaleHoldingsPath`：灰度持仓接口路径
  - `CoinUnlockListPath`：代币解锁列表接口路径
  - `CoinUnlockListTopCount`：代币解锁列表默认保留条数

## 联动关系
- `Shared/Protocol`：复用统一请求/响应协议封装。
- `Client/src/components/indicator`：前端指标中心可直接消费 `indicator.*` 协议（最新值、历史、实时频道缓存），其中 ETF 卡片在同一组件内通过 `scope.asset` 切换 BTC / ETH / SOL / XRP，合约足迹图与大户账户数多空比卡片在同一组件内切换 BTC / ETH，灰度持仓卡片展示总持仓概览与明细列表，代币解锁卡片组合读取列表与单币详情并支持点击切换币种，Hyperliquid 卡片在同一组件内切换鲸鱼提醒、鲸鱼持仓、持仓排行、用户持仓、钱包持仓分布与钱包盈亏分布六组数据。
- 指标中心布局口径：基础图表类卡片（贪婪恐慌、ETF、多空比、资金费率、OI、强平、稳定币）默认使用 `1x1`；灰度持仓默认 `1x2`；代币解锁、交易所资产、Hyperliquid、爆仓热力图、合约足迹图默认 `2x2`。
- 指标中心交互口径：每张卡片右下角提供拖拽手柄，宽高按当前响应式网格单元吸附；拖拽结果保存在浏览器本地，刷新后继续沿用；当屏幕列数收缩时，跨列宽度会自动收敛到当前可用列数。
- 合约足迹图前端渲染口径：单卡片内切换 `BTC / ETH`，上半区展示最近若干根 15 分钟 K 线净主动买卖额，下半区展示最新一根 K 线的价格桶主动买卖分布。
- 爆仓热力图前端渲染口径：主图使用 `liquidationLeverageData + yAxis + priceCandlesticks` 叠加“热力图 + K 线”；右侧按动态价格区间等分 100 桶，输出上涨/下跌最大痛点与当前价上下累计清算（订单簿式统计）。
- 交易所资产前端渲染口径：单卡片内切换三组数据，分别展示交易所资产明细、交易所余额排行、交易所余额趋势；三组数据均通过 `indicator.latest.get` 获取，不新增专用前端接口。
- Hyperliquid 前端渲染口径：单卡片内切换六组数据，其中持仓排行默认读取 `scope.symbol=BTC`，用户持仓默认读取 `scope.userAddress=0xa5b0edf6b55128e0ddae8e51ac538c3188401d41`；六组数据均通过 `indicator.latest.get` 获取，不新增专用前端接口。
- 代币解锁前端渲染口径：单卡片左侧展示解锁列表，右侧展示选中币种的流通/锁仓概况、分配结构和未来解锁计划；详情继续通过 `indicator.latest.get` 获取，不新增专用前端接口。
- `AdminBroadcast`：可基于 `indicator_refresh_logs` 增加运维看板与告警。

## 扩展新指标流程（建议）
1. 在 `indicator_definitions` 增加指标定义。
2. 在 `config_json` 声明 `source` 与 `fields` 字段目录（优先先定义字段路径）。
3. 新增采集器并实现 `IIndicatorCollector`（协议拉取、内部计算、自定义混合均可）。
4. 在采集器里输出“原始值 + 派生信号”，保证 latest/history 字段口径一致。
5. 在 `Program.cs` 注册采集器。
6. 若需额外展示形态，前端补充对应 `shape` 渲染组件。
