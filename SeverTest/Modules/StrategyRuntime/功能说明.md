# StrategyRuntime 模块功能说明

## 功能概述
- 策略运行时加载与生命周期管理。
- 通过租约机制保证单实例执行与接管。
- 后台服务负责运行时同步与预热。
- `testing` 状态支持无 API Key 重载：实例可在重启/接管后继续由运行时托管。
- 租约周期异常会触发安全降级：停止本节点已托管策略，降低多节点重复执行风险（当前阈值为首轮异常即触发）。

## 主要入口
- 运行与租约：`Application/StrategyRuntimeHostedService.cs`、`Application/StrategyRuntimeLeaseHostedService.cs`
- 加载与归属：`Application/StrategyRuntimeLoader.cs`、`Application/StrategyOwnershipService.cs`
- 启动：`Application/StrategyRuntimeBootstrapHostedService.cs`

## 联动关系
- `ExchangeApiKeys`：运行实例读取 API Key 信息。
- `MarketStreaming`：运行时依赖行情数据。
- `StrategyEngine`：实际策略执行引擎。

## 变更记录
- 2026-02-23：运行时消费改为单 worker 拉取行情任务，避免外层 worker 与引擎内层并行叠加导致高压线程竞争。
- 2026-02-23：新增租约连续异常安全降级机制，连续失败达到阈值后自动下线本节点托管策略。
- 2026-02-23：安全降级阈值收紧为首轮异常触发，缩短租约异常期间的双节点重复执行窗口。
