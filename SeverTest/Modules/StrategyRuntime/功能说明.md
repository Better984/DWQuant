# StrategyRuntime 模块功能说明

## 功能概述
- 策略运行时加载与生命周期管理。
- 通过租约机制保证单实例执行与接管。
- 后台服务负责运行时同步与预热。
- `testing` 状态支持无 API Key 重载：实例可在重启/接管后继续由运行时托管。
- 租约周期异常会触发安全降级：停止本节点已托管策略，降低多节点重复执行风险（当前阈值为首轮异常即触发）。
- 租约续租与策略同步已拆分为独立循环：续租不再被全量同步耗时阻塞，降低单机大策略量场景下的租约过期风险。
- 同步链路新增轻量查询与首轮水位机制：首轮同步仅记录更新时间水位，不再对已加载策略做重复全量重载。
- 新增实盘分布式节点链路：核心节点可向实盘节点分发 `upsert/remove` 指令，实盘节点仅运行实盘相关模块并执行策略增删。

## 主要入口
- 运行与租约：`Application/StrategyRuntimeHostedService.cs`、`Application/StrategyRuntimeLeaseHostedService.cs`
- 加载与归属：`Application/StrategyRuntimeLoader.cs`、`Application/StrategyOwnershipService.cs`
- 启动：`Application/StrategyRuntimeBootstrapHostedService.cs`
- 分布式实盘：`Application/LiveTradingWorkerGateway.cs`、`Application/LiveTradingWorkerDispatchService.cs`、`Application/LiveTradingWorkerClientHostedService.cs`

## 联动关系
- `ExchangeApiKeys`：运行实例读取 API Key 信息。
- `MarketStreaming`：运行时依赖行情数据。
- `StrategyEngine`：实际策略执行引擎。
- `StrategyManagement`：用户状态变更/删除触发实盘分布式指令下发（`upsert/remove`）。

## 配置选项
- `LiveTradingOptions`（`LiveTrading` 配置段）：`ShardCount` 控制行情任务消费分片数，1 = 单 worker，大于 1 = 键分片并行。

## 变更记录
- 2026-02-27：实盘模式切换为“任务到来即算”的极简链路，`StrategyRuntimeHostedService` 停止启动指标独立 worker，仅保留策略消费 worker；指标在策略执行前按任务去重并行刷新。
- 2026-02-26：修复单机大策略量场景租约抖动问题：`StrategyRuntimeLeaseHostedService` 改为“续租循环 + 同步循环”并行执行；续租支持并发处理；同步改为轻量头查询 + 按需回表加载，避免 `testing` 大量策略下出现“已释放/已接管”频繁震荡。
- 2026-02-26：Phase 3 键分片并行——新增 `KeyShardedStrategyConsumer`，按 exchange|symbol|timeframe 哈希分片消费，异键并行、同键串行；通过 `LiveTradingOptions.ShardCount` 配置化控制。
- 2026-02-26：`StrategyRuntimeHostedService.ExecuteAsync` 新增指标自动更新 worker（历史方案），该路径已在 2026-02-27 极简版本停用。
- 2026-02-25：新增实盘分布式节点协议与工作端客户端（`live-trading-worker`），支持主节点分发策略增删操作。
- 2026-02-23：运行时消费改为单 worker 拉取行情任务，避免外层 worker 与引擎内层并行叠加导致高压线程竞争。
- 2026-02-23：新增租约连续异常安全降级机制，连续失败达到阈值后自动下线本节点托管策略。
- 2026-02-23：安全降级阈值收紧为首轮异常触发，缩短租约异常期间的双节点重复执行窗口。
