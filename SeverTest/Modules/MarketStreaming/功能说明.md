# MarketStreaming 模块功能说明

## 功能概述
- 行情订阅与推送，提供价格与 K 线的实时流。
- 维护订阅关系并周期广播行情数据。
- 对外提供价格/行情查询接口。
- 行情任务支持多消费者订阅，避免竞争消费导致丢任务。
- 价格订阅服务支持“部分可用降级”：单个交易所初始化失败不会阻断整体启动，仅当全部交易所失败时才中止订阅。
- 提供 `IsExchangeReady` 用于策略侧检查交易所行情是否就绪。
- 行情广播发送复用连接级串行发送能力，避免与 WebSocket 响应链路并发发送产生冲突。
- 启动阶段会等待“订阅前置检查完成 + 首个价格入缓存”，价格推送未就绪时阻断系统就绪。
- 行情广播会按“活跃用户 + 当前订阅符号”清理 `_lastSent` 缓存，避免缓存键长期只增不减。
- 订阅队列在高压下改为“快速写入失败即拒绝（DropWrite）+ 收线任务关键补写”，优先保障 `isBarClose=true` 的关键任务不被静默挤掉。
- 行情主队列写入失败不再阻断订阅分发；收线任务会触发主队列关键补写，降低高压下关键任务漏发风险。
- 关键补写路径采用“有界补写队列 + 固定后台 worker”模型，避免高压下按失败次数创建 `Task.Run` 带来的调度抖动。

## 配置选项
- `MarketDataQueryOptions`：实时行情缓存长度与单次拉取上限。
- `RedisKeyOptions`：订阅关系在 Redis 的 Key 前缀。

## 主要入口
- 控制器：`Controllers/PriceController.cs`
- WebSocket 处理器：`Application/MarketSubscribeHandler.cs`、`Application/MarketUnsubscribeHandler.cs`
- 服务：`Application/MarketDataEngine.cs`、`Application/ExchangePriceService.cs`、`Application/MarketTickerBroadcastService.cs`、`Application/KlineCloseListenerService.cs`
- 订阅存储：`Infrastructure/RedisMarketSubscriptionStore.cs`、`Infrastructure/InMemoryMarketSubscriptionStore.cs`

## 联动关系
- `WebSockets`：通过消息处理器对接订阅/取消订阅，并向连接推送行情。
- `MarketData`：为实时 K 线接口提供数据来源。
- `StrategyEngine`/`StrategyRuntime`/`TradingExecution`/`Positions`：提供行情与价格能力。

## 变更记录
- 2026-02-27：修复行情队列压力告警计数在 `DropWrite` 下可能虚高的问题；`当前` 指标改为容量上限饱和计数，避免出现明显超出 `容量` 的告警值。
- 2026-02-02：修复 1m 收线任务时间戳使用上一根已收线 K 线，避免误取新开盘数据。
- 2026-02-23：行情广播发送切换为连接级串行发送，降低高并发下的 WebSocket 发送冲突风险。
- 2026-02-23：价格推送就绪纳入启动阻断，避免“系统已启动但价格推送不可用”的假就绪状态。
- 2026-02-23：行情广播增加 `_lastSent` 用户级/符号级回收，修复长期运行内存累积风险。
- 2026-02-23：交易所初始化改为“部分可用降级启动”，移除固定三家成功的硬性前置条件。
- 2026-02-23：订阅通道由 `DropOldest` 调整为 `DropWrite`，并新增收线任务补写机制，降低高压下关键收线任务丢失风险。
- 2026-02-23：行情主队列新增收线关键补写，且主队列写入失败时继续执行订阅分发，避免单点队列压力放大为全链路漏任务；配置为 `DropOldest/DropNewest` 时会自动校正为 `DropWrite`。
- 2026-02-23：关键补写改为固定后台 worker 消费有界补写队列，减少高压下 `Task.Run` 频繁调度造成的线程池竞争。
