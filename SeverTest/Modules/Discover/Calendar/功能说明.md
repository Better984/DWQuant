# Discover Calendar 功能说明

## 模块定位
- 提供 Discover 页面三类日历数据：
  - 央行活动（Central Bank Activities）
  - 财经事件（Financial Events）
  - 经济数据（Economic Data）
- 负责上游拉取、去重入库、内存热缓存和对外协议查询。

## 分层结构
- `Controllers/`：协议入口与鉴权校验。
- `Application/`：刷新编排、初始化补齐、内存缓存管理。
- `Infrastructure/`：数据库表结构与查询/UPSERT。
- `Domain/`：日历实体、查询模型、响应模型。
- `Sql/`：当前表结构与去重规则说明。

## 刷新与补齐策略
- 后台任务按 `DiscoverCalendar.PollIntervalSeconds` 固定间隔执行。
- 为避免上游默认只返回当天数据，服务端在首页刷新时会直接携带 `start_time/end_time` 发起单次区间请求，范围为“当天 00:00 到未来 N 天 23:59:59.999”：
  - 默认开启：`FutureWindowProbeEnabled=true`
  - 默认范围：`FutureWindowDays=7`
- 单次区间请求拿到的数据会直接按 `dedupe_key` 去重后统一 UPSERT 入库。
- 只有初始化补历史数据时，才会继续按分页请求更早的数据；常规刷新不再为未来窗口重复打多次请求。

## 存储与去重
- 三类日历分别独立建表：
  - `coinglass_calendar_central_bank_activities`
  - `coinglass_calendar_financial_events`
  - `coinglass_calendar_economic_data`
- 去重键：`calendar_name + publish_timestamp + country_code(+country_name)` 计算 SHA256。
- 入库策略：`ON DUPLICATE KEY UPDATE`，保留原有 `id`，更新事件字段与 `updated_at`。
- 内存缓存仅保留最近 `MemoryCacheMaxItems` 条，数据库保留全量历史。

## 对外协议
- `discover.calendar.central-bank.pull`
- `discover.calendar.financial-events.pull`
- `discover.calendar.economic-data.pull`
- 支持首次、增量、历史、时间区间四种拉取方式。

## 联动模块
- `Indicators/Infrastructure/CoinGlassClient`：复用 HTTP 请求与鉴权头。
- `Accounts/Application/AuthTokenService`：复用登录态校验。
- `Shared/Options`：复用日历与 CoinGlass 配置绑定/校验。
