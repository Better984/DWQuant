# Discover 模块功能说明

## 模块定位
- Discover 已拆为两个子模块，互相独立维护：
  - `Feed`：新闻（Article）+ 快讯（Newsflash）
  - `Calendar`：央行活动 + 财经事件 + 经济数据
- 两个子模块仅复用必要的公共依赖（CoinGlassClient、JWT 校验、协议封装）。

## 数据源说明
- 当前接入为开发阶段聚合商代理（非官方源，俗称“盗版聚合商”）。
- 仅用于开发联调与功能验证。
- 正式上线前必须切换为官方 CoinGlass 数据源。

## 目录结构
- `Feed/Controllers`、`Feed/Application`、`Feed/Infrastructure`、`Feed/Domain`、`Feed/Sql`
- `Calendar/Controllers`、`Calendar/Application`、`Calendar/Infrastructure`、`Calendar/Domain`、`Calendar/Sql`
- `Sql/README.md` 仅保留总入口说明，不再存放混合建表脚本。

## Feed（资讯）能力
- 后台轮询：按 `DiscoverFeed.PollIntervalSeconds` 固定间隔拉取（默认 7 秒）。
- 去重规则：`title + release_time + source` 计算 SHA256。
- 入库策略：`INSERT IGNORE`，重复数据自动忽略。
- 存储策略：数据库全量保留，内存最多 `MemoryCacheMaxItems`（默认 1000）。
- 拉取方式：首次、增量（`latestId`）、历史（`beforeId`）。

## Calendar（日历）能力
- 后台轮询：按 `DiscoverCalendar.PollIntervalSeconds` 固定间隔拉取（默认 600 秒）。
- 上游补齐：首页刷新直接携带 `language/page/per_page + start_time/end_time` 一次性拉取“当天 00:00 到未来 N 天 23:59:59.999”的区间数据，避免为补齐未来事件重复请求上游。
- 去重规则：`calendar_name + publish_timestamp + country_code(+country_name)` 计算 SHA256。
- 入库策略：`UPSERT`（`ON DUPLICATE KEY UPDATE`）。
- 存储策略：数据库全量保留，内存最多 `MemoryCacheMaxItems`（默认 1000）。
- 拉取方式：首次、增量（`latestId`）、历史（`beforeId`）、区间（`startTime/endTime`）。

## 当前项目配置（appsettings）
- `CoinGlassModules.CalendarEnabled=true`，已启用日历模块入口。
- `DiscoverCalendar.Enabled=true`，日历服务开启。
- `DiscoverCalendar.PollIntervalSeconds=86400`，按 1 天刷新一次。

## 初始化策略
- 启动先读本地库灌内存。
- 若表为空，按 `page/per_page` 分页拉取补齐，直到无更多数据或达到最大页数。
- Feed 与 Calendar 的补齐页数分别由各自配置控制，互不影响。

## 模块入口开关（CoinGlassModules）
- 在 appsettings 的 `CoinGlassModules` 中可统一配置是否启用：
  - `CalendarEnabled`：发现页日历（与 `DiscoverCalendar.Enabled` 同时为 true 时才会拉取）
  - `FeedEnabled`：发现页快讯/资讯（与 `DiscoverFeed.Enabled` 同时为 true 时才会拉取）

## 配置项
- 资讯配置：`DiscoverFeed.*`
  - 关键项：`Enabled`、`PollIntervalSeconds`、`ProviderPerPage`、`InitBackfillMaxPages`
- 日历配置：`DiscoverCalendar.*`
  - 关键项：`Enabled`、`PollIntervalSeconds`、`ProviderPerPage`、`InitBackfillMaxPages`、`FutureWindowProbeEnabled`、`FutureWindowDays`

## 联动模块
- `Indicators/CoinGlassClient`：复用 HTTP 鉴权与请求封装。
- `Accounts/AuthTokenService`：复用 Token 校验。
- `Shared/Protocol`：复用统一协议结构和错误码。
