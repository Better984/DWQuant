# 2026-02-17 风险说明 04：回测 Worker WebSocket 消息无大小上限

## 是什么
回测核心节点与算力节点通信时，文本消息读取使用 `MemoryStream` 聚合，但没有消息体大小上限保护。

代码证据：
- `SeverTest/Modules/Backtest/Application/BacktestWorkerGateway.cs:186`
- `SeverTest/Modules/Backtest/Application/BacktestWorkerGateway.cs:205`
- `SeverTest/Modules/Backtest/Application/BacktestWorkerClientHostedService.cs:373`
- `SeverTest/Modules/Backtest/Application/BacktestWorkerClientHostedService.cs:392`

对照：主用户 WebSocket 已有大小限制。
- `SeverTest/Modules/WebSockets/Application/WebSocketHandler.cs:222`

## 为什么
无上限读取意味着异常节点或恶意输入可以持续推高 `MemoryStream` 占用。该风险是单请求即可触发的“尖峰型”内存风险。

长期运行下会出现：
- 单条大包导致瞬时内存暴涨。
- GC 无法及时回收时可能直接 OOM。
- 核心回测链路被拖垮，影响任务调度和结果回传。

## 怎么做
1. 在 Worker Gateway 和 Worker Client 两端统一增加 `MaxMessageBytes`。
2. 超限后立即关闭连接并记录审计日志。
3. 对 JSON 反序列化前做长度校验，防止大包进入业务层。
4. 配置项接入 `BacktestWorker`，支持环境分级。
5. 增加监控：超限次数、平均消息大小、P99 消息大小。
