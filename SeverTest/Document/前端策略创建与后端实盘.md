# 前端策略创建与后端实盘

> 目的：说明前端“创建策略”的配置流程，以及后端实盘运行的实际链路（以当前代码实现为准）。

---

## 一、前端创建策略流程（Strategy Editor）

### 1) 入口与整体结构

- 入口在策略模块页签“创建策略”，用户点击自定义创建后打开编辑流程。
- 点击“自定义创建”后先弹出确认提示（是否跳转创建策略页面），点击确认后进入全屏工作台。
- 编辑流程由 `StrategyEditorFlow` 统一编排。
- 创建流程默认进入“全能指标策略工作台”（全屏）：
  - 工作台通过 Portal 挂载到 `document.body`，作为独立弹层界面，脱离主容器布局与滚动上下文。
  - 顶部长条：左侧返回按钮，右侧设置按钮。
  - 首屏以居中面板列出“交易所 + 币种（带图标）+ 周期”供用户选择，确认后展开工作区。
  - 默认选择为“币安 + 5 分钟”。
  - 当“新建策略”没有 `initialConfig` 时，工作台会自动注入示例策略：预置 `EMA(15)`、`SMA(50)` 两个指标，并在开多容器生成 `CrossUp(EMA15, SMA50)` 条件，避免首屏空白。
  - 左侧：上方 K 线图（`klinecharts`），下方本地回测仪表盘（基于当前策略配置实时刷新统计）。
  - 右侧左块：已选指标面板 + 操作符快捷分类展示（支持拖拽到条件位快速改写）。
  - 右侧右块：策略条件容器（开多/开空/平多/平空）+ 筛选器（开多筛选器/开空筛选器）。
  - 顶栏左侧新增“编辑模式 / 回测模式”切换：
    - 编辑模式：保持原有 5 区块工作台布局（K线、回测、已选指标、快捷操作符、策略条件编辑器）。
    - 回测模式：隐藏“已选指标 / 快捷操作符 / 策略条件编辑器”，形成“左上 K 线 + 左下回测仪表盘 + 右侧仓位列表”的双列布局。
    - 回测模式下右侧仓位列表固定占主工作区约 `1/4` 宽度，左侧（K线+仪表盘）占约 `3/4`。
  - 右侧左块（`已选指标` 与 `K线字段`）在未手动拖拽高度分隔条时为自动高度：按内容自适应填充，最大不超过视口高度的 `2/3`。
  - 当右侧左块被压缩或内容超出时，面板与字段区会显示滚动条，保证可持续滚动选择；用户拖拽分隔条后进入手动高度模式。

### 1.1) 工作台本地回测预演（新增）

- 工作台仪表盘不再仅依赖“本地时钟 + Tick”占位，而是直接执行前端本地回测预演。
- 工作台回测预览中的“仓位列表”支持点击联动 K 线：
  - 点击仓位后，图表自动聚焦到该仓位时间区间（未平仓仓位使用“最新样本时间 + 10 根K线”作为可视化终点）。
  - 在图上自动绘制止盈止损风险收益区间（`riskRewardLong` / `riskRewardShort`），并按仓位多空方向展示。
- 工作台回测预览“仓位列表”支持全量预览模式（列表头部开关）：
  - 仓位列表改为全量显示（不再仅保留最近 6 条）。
  - 普通预览：手动点击单条仓位联动图表（单笔绘制）。
  - 全量预览：列表滚动与 K 线可视区双向联动；K 线按可见窗口批量绘制仓位风险收益区间（仅渲染可见范围并限制绘制数量，避免卡顿）。
  - 回测预览区中“回测摘要 / 仓位列表”支持中间横向拖拽调宽；默认仓位列表占比约 `1/3`，摘要占比约 `2/3`。
  - 仪表盘标题右侧直接展示“回测进度（已处理/总样本 + 百分比）”，替代原先摘要区内的独立进度块，便于首屏直读。
  - 资金曲线点位缓存从固定小窗口改为按回测样本量动态扩容（带上限保护），避免 30 天回测仅显示最近 1 天的问题，默认覆盖完整回测区间。
  - 新增 ECharts 收益资金曲线：在图上标注“最大连胜位置/次数”“最大连败位置/次数”与“最大回撤位置/百分比”。
  - 新增 ECharts 胜负饼图：按“盈利平仓/亏损平仓”双分区展示占比，并在图下突出显示“开仓总数量”。
  - 新增“盈亏日历分析”模块：支持 `月 / 周 / 日` 三种粒度切换与前后翻阅；每个格子展示 `总盈亏 / 开仓单数 / 胜率`，用于联动分析资金曲线与胜负分布。
  - 新增“时段盈利分析”模块：支持按 `开仓时间 / 平仓时间` 切换统计口径，支持 `小时级 / 日级` 粒度切换；提供“持仓周期点阵图（X=持仓时长，Y=收益率）”、时段收益胜率分布图、日级与小时级明细表，并输出“盈利最快持仓段/快速亏损持仓段/高胜率时段/明显亏损时段”总结。
  - 在普通/全量预览中，当前选中的仓位对应风险收益绘图区会进入“选中态”高亮（更高对比度），便于在大量区间中快速定位当前仓位。
  - 新增“联动开关（图标+Toggle）”：可独立控制“滚动仓位列表 -> K线跟随”与“拖动K线 -> 列表跟随”双向联动，用户可按偏好关闭联动并手动浏览。
  - 联动开关默认关闭（需要用户主动开启）。
  - 图表中的仓位风险收益绘图区（`riskRewardLong/riskRewardShort`）为强锁定模式：不允许拖动，右键不可删除。
- 本地回测输入：
  - 当前 K 线历史（由 `StrategyWorkbenchKline` 拉取并回传）。
  - 当前已选指标（talib-web 计算结果）。
  - 条件容器（开多/平多 + 开多筛选器）。
  - 风险参数（止盈、止损、杠杆、开仓量）。
- 本地回测执行口径与后端对齐：
  - 条件方法实现与后端 `ConditionMethodRegistry` 保持一致（比较、交叉、区间、趋势、统计、带宽）。
  - 评估顺序保持一致：Cross 条件优先、Required 先剪枝、Optional 达标后提前结束。
  - 动作执行口径保持一致：`MakeTrade` -> 内存撮合、收盘价成交、止盈止损先于逻辑动作。
  - 当前与后端实盘一致，执行多空分支（`Entry.Long` / `Entry.Short` / `Exit.Long` / `Exit.Short`）。
- 仪表盘实时展示：
  - 交易笔数、胜率、累计收益、收益率、最大回撤、浮动盈亏、当前仓位、最新事件等。
  - 若指标未就绪或参数不足，会显示阻断/等待状态，不执行错误预演。

### 2) 指标选择与生成

- 点击“新增指标”打开指标生成器，载入 `talib_indicators_config.json`（当前为 160 项 TA 指标，前后端同源配置）。
- 指标生成后会形成 `refType: "Indicator"` 的配置（包含 input / params / output / offsetRange / calcMode）。
- 工作台左上 K 线图会按当前“交易所 + 币种 + 周期”调用后端 `marketdata.kline.history` 拉取历史 K 线并渲染。
- 工作台图表与“已选指标”实时联动：新增/修改/移除指标后，图表会自动挂载/更新/移除对应 TA 指标，并带入配置参数与输入源（如 `SMA(30)`、`MACD(12,26,9)`）。
- 策略编辑器采用“单策略单周期”：指标配置界面不再提供指标周期选择，指标周期默认跟随策略交易周期。
- 当用户修改策略交易周期时，已选指标会同步切换到新周期，条件引用无需手动重配。
- 生成的指标会被加入“已选指标”列表，供条件编辑时引用。
- 指标生成器的筛选组件与图表“指标选择”保持一致：支持 `全部/主图/副图` 与 `全部分类/分类标签` 双层筛选，并支持关键词检索联动。
- 指标生成器配置弹窗支持“输出项”显式选择（多输出指标不再固定首输出）。
- 指标生成器配置弹窗支持 `Periods` 输入槽（如 `MAVP`），并使用命名输入编码（示例：`Real=Close;Periods=High`）传递到后端。
- 图表端技术指标（KLine）使用 `talib-web` + `klinecharts.registerIndicator` 动态注册：
  - 启动时读取 `talib_indicators_config.json` 与 `talib_web_api_meta.json`
  - 自动完成后端 `code/method` 到 talib-web 函数映射
  - `klinecharts` 使用仓库内本地源码 `Client/vendors/klinecharts_local`（包含魔改能力）
  - Vite 解析策略为“优先 `vendors/klinecharts_local/dist/index.esm.js`，缺失时回退 `vendors/klinecharts_local/src/index.ts`”，避免新设备未构建 dist 时无法启动
  - 统一注册为 `ta_****`（例如 `ta_MACD`），并根据输出 hint 自动选择 line/bar/circle 绘制
  - 支持全量 160 项配置指标（主图/副图按分组自动归类）
  - 分类中文化：支持配置字段 `indicator_type_cn` / `group_cn`，未配置时自动将标准英文分类映射为中文展示
  - 指标展示名中文化：优先使用 `name_cn`，缺失时回退 `name_en/abbr_en/code`
  - 指标参数编辑弹窗按元数据驱动：支持 `Integer/Double/MAType(enum)` 参数类型
  - 参数默认值优先使用 `talib_web_api_meta.json` 的 `defaultValue`（如 MACD 的 12/26/9），缺失时再回退配置默认值
  - 指标输入源编辑支持 `Open/High/Low/Close/Volume/HL2/HLC3/OHLC4/OC2/HLCC4`，并通过 `extendData.taInputMap` 注入图表计算
- 图表侧参数与输入源约束与后端 `IndicatorKey(input/params/output/calcMode)` 一致，避免“前端可配但后端无效”
  - 图表交互改为“独立指标选择弹窗”：支持主/副图筛选、分类筛选、关键字搜索和副图一键关闭
  - 副图指标选择改为“多选叠加”，新增副图时不再替换已有副图
  - 本地魔改 `klinecharts` 的多设备同步与 `dist` 构建流程，见 `SeverTest/Document/KLineCharts本地源码同步指南.md`

### 3) 条件容器与条件组

- 默认 4 个条件容器：开多 / 开空 / 平多 / 平空。
- 每个容器最多 3 个条件组；每组最多 6 个条件判断。
- 条件支持：启用/必需、比较/交叉/区间/持续/统计等方法，支持多参数与窗口长度（N）等附加参数。
- 取值支持：左右值/第三参数可为指标字段或数值常量，参数字段通过 `param` 传递（如窗口长度 N）。
- 条件编辑弹窗改为“先选类型再选条件”，并按模板仅展示必要参数，减少新手困惑。
- 条件编辑新增实时合法性约束：
  - 选择左值后，右值/第三值会按规则动态隐藏非法选项（主图/副图混比、自身对自身等）。
  - 指标配对比较时禁止跨周期组合（例如 `SMA 1m` 对比 `SMA 5m` 会被实时拦截与隐藏）。
  - 当右值/第三值切到“字段”但当前无适配对象时，UI 不再自动回退到“数值”，并显示“已选参与值：xxx，当前没有适配的交互对象”提示。
  - 同组内重复条件会被拦截，防止无效堆叠。
  - 提交前会进行全量条件合法性检查与数量限制检查（每组 ≤ 6、每集合 ≤ 3 组、总条件 ≤ 108）。
- 工作台拖拽编辑增强：
  - 已选指标项使用前端固定色板（常用指标优先不重复色），同指标删后再加仍尽量保持同色。
  - 指标输出值显示为可拖拽标签，颜色沿用指标主色并通过透明度区分输出项。
  - 条件中引用到指标输出时，左值/右值标签与源指标“具体输出项”保持一致颜色与深浅（如 `MACD line`、`signal` 各自深浅不同），便于快速识别引用关系。
  - 支持悬停双向定位：悬停指标输出会触发所有引用该值的条件标签抖动；悬停条件中的引用值也会反向触发指标输出抖动。
  - 抖动联动仅在“存在真实引用关系”时触发：若某输出当前未被任何条件引用，悬停时不触发抖动，避免无效干扰。
  - 抖动动效为短时触发（仅抖动数次），不做持续循环，降低视觉噪音。
  - 条件项改为“两行结构”：第一行展示“判断类型 - 操作符”，第二行为“左值 / 操作符 / 右值”三槽位。
  - 支持将“指标输出值”拖到左值/右值槽位；支持将“操作符”或“分类”拖到中间槽位快速替换条件方法。
  - 支持同一条件内左右值互换：当左右任一侧已配置字段时，可将该值拖到另一侧进行“交换/挪位”；不允许跨条件、跨条件组直接互换。
  - 若“右值字段替换”后触发语义冲突（例如主图指标对比副图指标），仍允许本次替换并自动将左值置为“未配置”；同时给出软提示，用户可继续编辑或直接 `Ctrl+Z` 撤回。
  - 拖拽命中策略使用 `pointerWithin`，仅鼠标精准落在可投放槽位时才触发修改，避免“附近释放也命中”。
  - 拖拽时增加跟随鼠标的浮动预览标签，提升拖拽过程可见性与定位反馈。
  - 拖拽中的源标签不再位移，仅显示鼠标跟随浮层，避免已选指标面板被拉宽并触发底部横向滚动条。
  - 浮层样式与被拖拽项保持一致，并以鼠标中心点为锚点显示，减少视觉偏移感。
  - 支持将 `K线字段`（Open/High/Low/Close/Volume 等）拖到“已选指标项”快速改输入源；若该指标已被条件引用，会弹出与编辑一致的确认面板。
  - 指标引用确认面板新增“修改且本次不再提示”按钮，可在当前会话跳过后续同类确认。
  - 支持将“快捷操作符分类”或“具体操作符”拖到条件容器/条件组，快速创建条件。
  - 拖到容器时：若无条件组自动创建条件组并创建条件；已有条件组则默认在首个条件组创建。
  - 拖到条件组时：在指定条件组创建条件，简化“先建组再建条件”流程。
  - 若快捷创建时自动补全参与值失败（如当前候选不足或语义冲突），仍会先创建条件，并将参与值置为“未配置”，避免阻断创建流程。
  - K线字段区新增“数值”拖拽项：可直接拖到条件右值，将右值切为数值并在外层槽位直接输入（无需再打开条件编辑弹窗）。
  - “数值”拖到指标项时：会进入指标参数快速编辑路径（参数如 MA 的周期值），并优先聚焦参数输入；若该指标无可编辑参数则给出软提示。
  - 若“数值拖拽改参数”目标指标已被条件引用，同样会进入引用确认流程后再允许继续修改。
  - 支持编辑历史（指标 + 条件）：用户可使用 `Ctrl+Z` 撤销、`Ctrl+Y` / `Ctrl+Shift+Z` 还原。
  - 撤销/还原会给出细粒度软提示，例如“撤销：新增指标 SMA 30”“还原：修改指标输入源 EMA 15: Close -> Open”“撤销：新增条件组/新增条件/修改条件右值数值”等。

### 4) 交易规则配置与生成 JSON

- 点击“生成配置”打开预览弹窗：可在“条件概览”和 JSON 之间切换。
- 交易规则配置内容包括：交易所、交易对、周期、单次开仓数量、最大持仓、杠杆、止盈/止损/移动止盈。
- 工作台左下“本地实时检测仪表盘”要求先填写止盈比例、止损比例、杠杆倍数、单次开仓数量；未完成时不启动实时检测。
- 运行时间配置：支持模板多选（如 A 股/美股），模板由服务端提供并带唯一模板 ID；自定义不交易/交易时段（自定义按小时精度）。
- 模板支持分钟级时间段（例如 09:30），并可挂载交易日历用于节假日/异常交易时段。
- 运行时间模板与时区由接口 `POST /api/strategy/runtime/templates` 获取（协议类型：`strategy.runtime.template.list`）。
- 默认策略：非运行时间仅禁止开仓，允许平仓/风控（可通过 `outOfSessionPolicy` 调整）。
- 交易规则配置区域随内容自适应，弹窗主体负责滚动；切换步骤会自动回到顶部。
- 交易规则步骤切换仅通过“下一步/上一步”按钮，滚轮不再自动切换步骤。
- 移动止盈止损参数在启用后显示，包含触发收益阈值与回撤触发比例。
- 最终生成 `StrategyConfig`：
  - `trade`: 交易规则
  - `logic`: 逻辑分支（entry/exit, long/short）
  - `runtime`: 运行时间配置（模板仅保存模板 ID 列表）

## 二、后端实盘运行链路（Server Runtime）

### 1) 策略创建入库

`POST /api/strategy/create`：
- 新建 `strategy_def`
- 新建 `strategy_version`（version_no=1）
- 新建 `user_strategy`（state=completed, visibility=private）
- 更新 `strategy_def.latest_version_id`
- 入库前执行双重校验：`StrategyRuntimeConfigValidator`（运行时） + `StrategyConditionConfigValidator`（条件合法性与规模限制）

涉及文件：
- `SeverTest/Modules/StrategyManagement/Controllers/StrategyController.cs`
- `SeverTest/Document/策略系统.md`

### 2) 运行状态切换（关键入口）

`PATCH /api/strategy/instances/{id}/state`：
- state=running / paused_open_position：解析 config_json -> 组装 StrategyDocument -> 加载为运行时策略 -> 注册到 `RealTimeStrategyEngine`
- state=paused / completed：从 `RealTimeStrategyEngine` 移除
- 可选传 `exchangeApiKeyId`：绑定该策略实例使用的交易所 API key
- 未传时会自动回填该交易所最新 API key（updated_at DESC）；无可用 API 时返回错误
- 若 API key 所属交易所与策略配置不一致，会自动创建新版本并同步 `trade.exchange`

涉及文件：
- `SeverTest/Modules/StrategyManagement/Controllers/StrategyController.cs`
- `SeverTest/Modules/StrategyEngine/Application/StrategyJsonLoader.cs`
- `SeverTest/Modules/StrategyEngine/Application/RealTimeStrategyEngine.cs`

### 3) 行情驱动与执行主链路

- `MarketDataEngine` 产出行情任务（MarketDataTask）。
- `StrategyRuntimeHostedService` 启动指标引擎与策略引擎 worker。
- `RealTimeStrategyEngine` 收到行情任务后：
  1) 刷新指标（IndicatorEngine）
  2) 策略运行时间门禁判定（runtime，模板 ID + 日历异常）
  3) 条件判断（ConditionEvaluator + ConditionMethodRegistry）
  4) 触发动作（ActionMethodRegistry -> IStrategyActionExecutor）

涉及文件：
- `SeverTest/Modules/StrategyRuntime/Application/StrategyRuntimeHostedService.cs`
- `SeverTest/Modules/StrategyEngine/Application/RealTimeStrategyEngine.cs`
- `SeverTest/Modules/StrategyEngine/Application/IndicatorEngine.cs`
- `SeverTest/Modules/StrategyEngine/Application/ConditionEvaluator.cs`
- `SeverTest/Modules/StrategyEngine/Domain/ConditionMethodRegistry.cs`
- `SeverTest/Modules/StrategyEngine/Domain/ActionMethodRegistry.cs`

### 4) 下单与持仓落地

- 当前默认执行器为 `QueuedStrategyActionExecutor`，会把 `StrategyActionTask` 入队。
- `TradeActionConsumer` 消费队列，调用 `CcxtOrderExecutor` 市价下单。
- 开仓成功后写入 `strategy_position`。

涉及文件：
- `SeverTest/Modules/StrategyEngine/Application/QueuedStrategyActionExecutor.cs`
- `SeverTest/Modules/StrategyEngine/Application/StrategyActionTaskQueue.cs`
- `SeverTest/Modules/TradingExecution/Application/TradeActionConsumer.cs`
- `SeverTest/Modules/TradingExecution/Infrastructure/CcxtOrderExecutor.cs`
- `SeverTest/Modules/Positions/Infrastructure/StrategyPositionRepository.cs`

### 5) 风控平仓（止盈/止损/移动止盈）

- `PositionRiskEngine` 周期性扫描未平仓位，结合 TP/SL/Trailing 触发平仓。
- 平仓后更新 `strategy_position` 状态。

涉及文件：
- `SeverTest/Modules/Positions/Application/PositionRiskEngine.cs`

## 三、当前实现注意点（与策略运行强相关）

1) **Short 分支未启用**
- `exit.short` / `entry.short` 在策略引擎中仍被注释，当前只执行 Long 分支。
- 涉及文件：`SeverTest/Modules/StrategyEngine/Application/RealTimeStrategyEngine.cs`

2) **启动时自动加载 DB 中运行策略**
- `StrategyRuntimeBootstrapHostedService` 启动时加载 `user_strategy` 中 `running/paused_open_position/testing` 的实例并注册到 `RealTimeStrategyEngine`。
- 会尝试自动回填 `exchange_api_key_id`；找不到可用 API key 的策略会跳过加载。
- `StrategyRuntimeHostedService` 仍只负责启动引擎 Worker。
- 运行策略也可通过 `PATCH /api/strategy/instances/{id}/state` 即时更新。
- 涉及文件：`SeverTest/Modules/StrategyRuntime/Application/StrategyRuntimeBootstrapHostedService.cs` `SeverTest/Modules/StrategyRuntime/Application/StrategyRuntimeHostedService.cs`

3) **前端“数值常量比较”与后端解析不完全对齐**
- 前端会生成 `refType: "Const"` 的右值；
- 后端仅识别 `Indicator`，其余走字段解析并回落到 K 线字段默认值。
- 涉及文件：`Client/src/components/StrategyEditorFlow.tsx` `SeverTest/Modules/StrategyEngine/Application/IndicatorValueResolver.cs`

4) **开仓上限执行语义**
- `TradeActionConsumer` 不再以“已有同向仓位”直接忽略开仓，而是在执行下单前按 `maxPositionQty` 校验：`当前同向持仓 + 本次开仓量` 超限即阻断；
- 被阻断事件会写入 `order_open_attempt`（含信号时间、信号价、上限值、当前持仓与本次开仓量），用于追溯“命中信号但未开仓”；
- `openConflictPolicy` 字段仍未单独生效，当前以 `maxPositionQty` 作为实际开仓阻断规则。
- 涉及文件：`SeverTest/Modules/TradingExecution/Application/TradeActionConsumer.cs` `SeverTest/Modules/Shared/Models/Strategy/StrategyModels.cs`

## 四、快速对照：前端生成与后端读取的配置结构

前端生成 `StrategyConfig`：
```
{
  "trade": {
    "exchange": "bitget",
    "symbol": "BTC/USDT",
    "timeframeSec": 60,
    "positionMode": "Cross",
    "openConflictPolicy": "GiveUp",
    "sizing": { "orderQty": 0.001, "maxPositionQty": 10, "leverage": 100 },
    "risk": { "takeProfitPct": 0.02, "stopLossPct": 0.01, "trailing": { ... } }
  },
  "logic": {
    "entry": { "long": { ... }, "short": { ... } },
    "exit": { "long": { ... }, "short": { ... } }
  },
  "runtime": {
    "scheduleType": "Template",
    "outOfSessionPolicy": "BlockEntryAllowExit",
    "templateIds": ["cn.a_share.regular"],
    "custom": {
      "mode": "Deny",
      "timezone": "Asia/Shanghai",
      "days": ["sat", "sun"],
      "timeRanges": [
        { "start": "00:00", "end": "00:00" }
      ]
    }
  }
}
```

后端解析并运行：
- `StrategyJsonLoader.ParseConfig()` -> `StrategyConfig`
- `RealTimeStrategyEngine` 使用 `trade + logic` 执行
- 运行时间模板由服务端模板库（数据库）与日历解析得到具体交易时段

涉及文件：
- `SeverTest/Modules/StrategyEngine/Application/StrategyJsonLoader.cs`
- `SeverTest/Modules/StrategyEngine/Application/RealTimeStrategyEngine.cs`
