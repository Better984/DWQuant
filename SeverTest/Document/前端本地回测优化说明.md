# 前端本地回测优化说明

## 目标
- 在策略工作台中实现“边回测边反馈”，避免用户长时间无感等待。
- 在频繁改参、频繁调整条件的场景下减少重复计算开销。

## 当前已落地优化
1. 分片实时回测
- 文件：`Client/src/components/strategy/localBacktestEngine.ts`
- 能力：`runLocalBacktestRealtime`
- 机制：
  - 将全量 K 线按 `chunkSize` 分片执行；
  - 每个分片完成后通过 `onProgress` 回调推送最新 summary；
  - 前端可实时显示仓位数、胜率、平均盈亏、检测进度、检测时间。

2. 指标序列缓存（内存）
- 文件：`Client/src/components/strategy/localBacktestEngine.ts`
- 机制：
  - 以“样本签名（bars）+指标配置签名（selectedIndicators）”作为 key；
  - 命中时直接复用 TALib 序列结果，跳过重复指标计算；
  - 采用有限容量（FIFO）控制内存增长。

3. 运行时编译缓存（内存）
- 文件：`Client/src/components/strategy/localBacktestEngine.ts`
- 机制：
  - 以“逻辑容器 + 过滤容器 + 方法定义”构建运行时签名；
  - 命中时复用 `RuntimeBundle`，减少条件运行时编译成本；
  - 采用有限容量（FIFO）控制内存增长。

4. 高编辑频率防抖
- 文件：`Client/src/components/strategy/StrategyWorkbench.tsx`
- 机制：
  - 回测触发前增加轻量防抖（默认 80ms）；
  - 避免用户连续输入时每次按键都触发完整重算。

5. 触发策略
- 文件：`Client/src/components/strategy/StrategyWorkbench.tsx`
- 当前触发源：
  - 回测参数变化（止盈止损、杠杆、开仓量、手续费、资金费率、滑点等）；
  - 多空条件变更；
  - 指标变更；
  - K 线数据变化（例如加载更多历史）。

6. 回测样本时间范围（新增）
- 文件：
  - `Client/src/components/strategy/StrategyWorkbenchKline.tsx`
  - `Client/src/components/strategy/StrategyWorkbench.tsx`
- 机制：
  - K线初始样本窗口由“固定720根”改为“按当前周期换算最近30天”；
  - 回测默认使用“最近30天”；
  - 参数面板新增“回测范围”设置，支持：
    - 最近30天（默认）
    - 自定义起止时间（`datetime-local`）；
  - 仪表盘实时显示“可用样本范围”和“当前生效范围”，便于确认本次回测实际使用的数据区间。

7. 条件容器启用状态兼容（新增）
- 文件：`Client/src/components/strategy/StrategyEditorFlow.tsx`
- 背景：
  - 历史配置中可能缺少 `branch.enabled` 字段，导致“开多/开空容器”被误判为禁用；
  - 用户即使新增了条件，也可能因为容器仍为禁用而无法触发开仓。
- 处理：
  - 解析历史配置时：当 `enabled` 缺失且存在条件组时，自动推断为启用；
  - 在工作台新增条件组、快速新增条件、保存条件时，自动将对应容器置为启用。

8. 回测阶段诊断日志（新增）
- 文件：
  - `Client/src/components/strategy/localBacktestEngine.ts`
  - `Client/src/components/strategy/StrategyWorkbench.tsx`
- 机制：
  - 引擎按阶段累计诊断计数（数据、筛选、条件命中、动作执行、失败原因、风控、最终结果）；
  - 每轮实时回测 summary 都附带 `diagnostics` 文本列表；
  - 仪表盘预览区新增“回测阶段诊断”展示；
  - 回测完成后在浏览器控制台输出最终诊断日志（`[本地回测诊断]`）。

9. 高级统计与明细汇总（新增）
- 文件：
  - `Client/src/components/strategy/localBacktestEngine.ts`
  - `Client/src/components/strategy/StrategyWorkbench.tsx`
  - `Client/src/components/strategy/StrategyWorkbench.css`
- 机制：
  - 本地回测 summary 新增高级统计字段：
    - `avgProfit`、`profitFactor`、`avgWin`、`avgLoss`
    - `sharpeRatio`、`sortinoRatio`
    - `annualizedReturn`、`calmarRatio`
    - `maxConsecutiveWins`、`maxConsecutiveLosses`
    - `avgHoldingMs`、`maxDrawdownDurationMs`
  - 引擎在分片执行过程中增量维护交易聚合与事件分布，避免每次进度回调做全量重算；
  - 新增资金曲线摘要（点数、最高/最低权益、区间最大盈利/亏损）；
  - 新增事件摘要（总数、首末时间、类型分布）与“最近事件”列表；
  - 仪表盘预览区同步展示高级指标、交易汇总、资金曲线汇总、事件汇总，保持实时刷新体验。

## UI 实时反馈
- 文件：`Client/src/components/strategy/StrategyWorkbench.tsx`
- 仪表盘预览区：
  - 左侧：基础指标 + 高级指标（收益、回撤、Sharpe/Sortino、年化/Calmar、连胜连败、持仓时长等）、进度条、阶段诊断、最近事件；
  - 右侧：实时仓位列表（最近 N 条）。

## 已知限制与后续方向
1. 多空分支“增量复用”尚未完成
- 当前每轮仍执行同一套主循环，仅复用指标与运行时编译结果。
- 后续可继续引入“分支依赖图 + 局部失效”机制。

2. 条件级命中缓存仍是单轮内缓存
- 当前 `evaluateCondition` 的 cache 主要用于单次回测过程。
- 后续可探索跨轮次复用，但需严格处理时间窗与参数变更失效策略。

3. 右侧列表采用“最近 N 条”单屏展示策略
- 目的：不引入滚动条、保持预览区稳定。
- 若需全量明细，可在后续加入“弹窗明细”或“导出”路径。
