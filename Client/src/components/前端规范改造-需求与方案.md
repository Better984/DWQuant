# 前端规范改造：需求理解与执行方案

## 一、需求理解

1. **规范命名**：去掉 SnowUI、DWQuant 等与业务无关或强品牌化命名，采用中性、通用的规范命名。
2. **组件分目录、按功能存放**：在 `components` 下按功能/模块划分子目录，便于维护和定位。
3. **整体更规范**：命名、目录、文档一致，使前端框架更清晰、可维护。
4. **文档同步**：参考已有文档（如 AGENTS.md、COMPONENTS.md、项目结构说明），改造后同步更新或删除过时文档。

## 二、命名规范（中性、通用）

| 类型 | 现状 | 调整后 |
|------|------|--------|
| CSS 类前缀 | `snowui-*` | `ui-*`（如 `ui-dialog`、`ui-button`） |
| 静态资源目录 | `assets/SnowUI/` | `assets/icons/`，子目录：`crypto`、`cex`、`country`、`head`、`icon`（通用图标） |
| 登录页 Logo 文件 | `snowui-logo-signin4.svg` | `logo-signin.svg` |
| 本地存储 key | `dwquant.systemNotifyPreference` | `app.systemNotifyPreference` |
| 测试/调试页 | `SnowUITest` + `UIComponentsTest` 两条路由 | 保留一个 UI 测试页 `UIComponentsTest`，路由 `/ui-test`；移除 SnowUITest 及 `/snowui-test` |

说明：npm 包 `@snowui-design-system/resource-react` 仅作为图标/资源来源保留，不体现为页面或类名品牌。

## 三、组件目录结构（按功能）

```
Client/src/components/
├── layout/          # 布局：Dashboard 等
├── strategy/        # 策略：StrategyModule、StrategyList、StrategyItem、各类 Strategy* Dialog、Condition*、TradeConfigForm 等
├── market/          # 行情：MarketModule、MarketChart、CryptoMarketPanel、TradingViewDatafeed、customOverlays 等
├── indicator/       # 指标：IndicatorModule、IndicatorGeneratorSelector
├── discover/        # 发现/资讯：DiscoverModule、WhatsOnRoadPanel
├── home/            # 首页：HomeModule
├── chat/            # 聊天：ChatModule
├── auth/            # 认证：AuthPage、SignIn4、ChangePassword
├── user/            # 用户：UserSettings
├── dialogs/         # 通用弹窗：AlertDialog、HistoricalDataCacheDialog（被多模块引用）
├── ui/              # 基础 UI 组件（Button、Dialog、Select、Avatar 等），保持现有文件，仅改类名
├── COMPONENTS.md    # 组件说明（同步更新路径与描述）
└── (根目录)         # 入口/测试：TestPage、UIComponentsTest、KlineChartsDemo、WsNotificationBridge 等保留在根或集中到 _dev 视情况定
```

说明：`ui/` 仅做类名前缀替换，不移动；业务组件按功能迁入对应子目录，并统一更新引用路径。
